Identity

Compound key

{
  'identity': { 
    'id': 'some_app',
    'originator_id': '339e0f20-383b-11e4-916c-0800200c9a66'
  },
  /* .. */
}

Properties

Immutable for the lifetime of a payload
Payloads with same identity refer to same app
Necessary because URI may change

Core Attributes

{
  'identity': { 
    'id': 'some_app',
    'originator_id': '339e0f20-383b-11e4-916c-0800200c9a66'
  },
  'attributes': {
    'uri': 'http://example.com',
    'timestamp': '2014-07-29T21:52:58+00:00',
    'share': true,
    'propagate': true,
    'use':{ /* .. */ },
    'require':{ /* .. */ }
  }
}

Extensible Attributes

{
  /* .. */
  'attributes': {
    /* .. */
    'use':{
      "1f2625c2-615f-11e3-bf13-d231feb1dc81":"UCLA Directory"
    }
  }
}

UUID-based to prevent namespace conflicts
Requirement levels regardless of if recognized

LTI ContentItem

"The content-item message is used to provide a mechanism for users to interact with a Tool Provider (TP) in the process of selecting one or more content items for insertion into the tool consumer."

LTI launch

From any LTI consumer
To app store as an LTI provider

ContentItem message

Posted back to LTI consumer

LTI ContentItem Envelope

<form action="http://consumer.example.com/add-app" method="post"
 encType="application/x-www-form-urlencoded">
  <input type="hidden" name="lti_message_type" 
                       value="ContentItemSelectionResponse" />
  <input type="hidden" name="lti_version" value="LTI-1p0" />
  <input type="hidden" name="content_items" value=".." />
  <input type="hidden" name="data" value="Some opaque TC data" />
  <input type="hidden" name="oauth_version" value="1.0" />
  <input type="hidden" name="oauth_nonce" value="..." />
  <input type="hidden" name="oauth_timestamp" value="..." />
  <input type="hidden" name="oauth_consumer_key" value="..." />
