Captions

Interoperability & Interpretation

Structure is critical, but it doesn't fix everything (javascript)

Accessibility elements provide information but can be used wrong (poor alt text)

We often rely on the assumption that proper structure will provide clear context (header overuse)

The Accessibility Technology Layer

Assistive Technology is implemented each having different ways of presenting information

Gray areas often take on the form of custom, heuristic solutions, making consistent translation of semantics more difficult

Requires developers to have to jump through hoops to manage multiple technologies, not unlike browser wars

Barriers to Accessibility

Standards lag behind, development of AT bottlenecked

Hard to machine test, many false positives

Poor implementation can make situations worse

Often includes header and footer
Should make sense independent of the page context

<section>

a generic document or application section... not a generic container element... appropriate only if the element’s contents would be listed explicitly in the document’s outline.

Usually shouldn't be used if no natural heading
Don't use it when you need solely for styles or scripts
Most generic of the sectioning elements

When use article, section or div?

article if the content would make sense on its own in a feed reader

section if the content is related

div otherwise

<nav>

section of a page that links to other pages or to parts within the page... only sections that consist of major navigation blocks are appropriate

Site navigation
Table of contents
Previous/next buttons or pagination
Search form
Breadcrumbs

Usually contains a list, but sometimes just buttons, forms or links
UAs may omit on initial rendering, and make it easy to skip to

<aside>

content that is tangentially related to the content around [it]... often represented as sidebars in printed typography

Pull quotes
Sidebars
Glossary
Blogroll
Advertisements

Not for parentheticals, as they're still part of the main flow
Within a sectioned element, tangential to section (like glossary)

<figure> and <figcaption>

unit of content... that is self-contained... typically referenced as a single unit from the main flow... that can be moved away from the main flow

Diagrams
Illustrations
Photos
Code examples
Audio and video

If the content is simply related and not essential, use aside
If the content is essential but its position isn’t, use figure

ARIA

Constituents of ARIA

Built to map to OS native accessibility APIs

  • Roles
  • Properties
  • States

Roles

widgets not otherwise available such as sliders, tabs and dialogs

  • Landmark
  • Structural
  • Widget

role="navigation"

Landmark Roles

Landmarks are regions on a page, common elements that have an expected set of behaviors or properties

Landmark Role Examples

role="list"

elements that are editable or submit content

role="main"

main content of a document

role="navigation"

menus / list of links

Structural Roles

Roles that provide content relevance

Structural Role Examples

role="img"

container for an image and its related parts such as a caption

role="list"

content ordered in a list, non-interactive

role="separator"

content separator, like an image based horizontal rule

Widget Roles

Widgets are roles that are more specific to the type of action being taken

Widget Role Examples

role="alert"

time sensitive information, will interrupt user

role="tabpanel", role="tablist"

content within a tab, part of a larger tablist

role="progressbar"

tick tock, tick tock

Properties

characteristics such as draggable, required, or an associated pop-up

Aria Properties Examples

aria-controls="elementid"

used with elements such as check boxes or radios, but not exclusively

aria-labelledby="labelid"

identifies the label related to the current element

aria-haspopup="true"

signifies the current element will create a pop up when activated

ARIA States

current interactive state such as disabled, hidden or live

ARIA States Examples

aria-selected="true"

used with elements such as check boxes or radios, but not exclusively

aria-hidden="true"

indicates contained elements are not visible / usable to any user

aria-disabled="true"

used on elements that can be seen, but not used / changed

(function (window, document) {

    function Shake() {
        this.hasDeviceMotion = 'ondevicemotion' in window;
        this.threshold = 15;
        this.lastTime = new Date();
        this.lastX = null;
        this.lastY = null;
        this.lastZ = null;

        if (typeof document.CustomEvent === "function") {
            this.event = new document.CustomEvent('shake', {
                bubbles: true,
                cancelable: true
            });
        } else if (typeof document.createEvent === "function") {
            this.event = document.createEvent('Event');
            this.event.initEvent('shake', true, true);
        } else { 
          return false;
        }
    }

    Shake.prototype.reset = function () {
        this.lastTime = new Date();
        this.lastX = null;
        this.lastY = null;
        this.lastZ = null;
    };

    Shake.prototype.start = function () {
        this.reset();
        if (this.hasDeviceMotion) { window.addEventListener('devicemotion', this, false); }
    };

    Shake.prototype.stop = function () {
        if (this.hasDeviceMotion) { window.removeEventListener('devicemotion', this, false); }
        this.reset();
    };

    Shake.prototype.devicemotion = function (e) {
        var current = e.accelerationIncludingGravity,
            currentTime,
            timeDifference,
            deltaX = 0,
            deltaY = 0,
            deltaZ = 0;

        if ((this.lastX === null) && (this.lastY === null) && (this.lastZ === null)) {
            this.lastX = current.x;
            this.lastY = current.y;
            this.lastZ = current.z;
            return;
        }

        deltaX = Math.abs(this.lastX - current.x);
        deltaY = Math.abs(this.lastY - current.y);
        deltaZ = Math.abs(this.lastZ - current.z);

        if (((deltaX > this.threshold) && (deltaY > this.threshold)) || ((deltaX > this.threshold) && (deltaZ > this.threshold)) || ((deltaY > this.threshold) && (deltaZ > this.threshold))) {
            //calculate time in milliseconds since last shake registered
            currentTime = new Date();
            timeDifference = currentTime.getTime() - this.lastTime.getTime();

            if (timeDifference > 1000) {
                window.dispatchEvent(this.event);
                this.lastTime = new Date();
            }
        }

        this.lastX = current.x;
        this.lastY = current.y;
        this.lastZ = current.z;
    };

    Shake.prototype.handleEvent = function (e) {
        if (typeof (this[e.type]) === 'function') {
            return this[e.type](e);
        }
    };

    var myShakeEvent = new Shake();
    myShakeEvent && myShakeEvent.start();

}(window, document));

Goals of IndieUI

Make it easier for web developers to author consistently usable interfaces that are input-agnostic and independent of a user's particular platform, hardware, locale, and preferences.

Enable every type of control in these interfaces to be programmatically determinable and controllable by both mainstream and alternate forms of user input, including assistive technologies.

Provide a clear path for web developers to smoothly transition from currently existing physical events to IndieUI events, during the period when implementations of IndieUI are incomplete.
