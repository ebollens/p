From Chaos to Order

A Trend Towards Micro

Originally, we wanted to make
one framework to rule them all

But to do everything for everyone,
we end up doing nothing for anyone

Looking to the Linux model,
do one thing and do it well

This means many, many awesome tools!

Keeping Tabs on It All

Getting a library...

Browse to a site that has the library
Download the library and add it to the project

Keeping a library up to date...

Check the site to see if there are any updates
If updated, download again and replace the old version

When you've got a bunch of libraries,
this adds a lot of overhead

Managing the Chaos

Git submodules

If using Git, they seem like an obvious choice
Submodules bound to repo only by commit ID
Submodules don't update with base repo pull
Merge resolution can trample on submodules

Package managers are a better choice

Node.js

But What About Front-End Assets?

Bower

What is Bower?

Straightforward package manager

All the usual operations
Semantic versioning
No language requirements

Dependency resolution

Discovery services

bower.io Directory
Bound to GitHub by default

Installation

npm install -g bower

Requires Node.js and NPM

Wrapping Bootstrap

Blockfile.rb  is not just a DSL - it's also true Ruby

[
  'buttons',
  'grid',
  'tables',
  'dropdown',
  'navbar',
  # etc...
].each do |name|
  block name do
    dependency twbs.route 'base'
    scss_file "stylesheets/bootstrap/_#{name}.scss"
  end
end

You can reopen blocks after they're defined as needed

block 'navbar' do
  dependency twbs.route 'navs'
  dependency twbs.route 'forms'
end
